# Repository Guidelines

## Project Structure & Module Organization
- `popup.tsx` hosts the Drift Boss launcher UI as a single React function component; update it for popup logic or Chrome API calls.
- `popup.css` defines the 400×300 popup look and hover effects; keep new styles scoped with class selectors.
- `assets/` stores the extension icon and marketing screenshots; add new images under `assets/screenshots/` to keep imports consistent.
- `build/` is generated by Plasmo (`chrome-mv3-dev` for hot reload, `chrome-mv3-prod` and `.zip` for release). Do not hand-edit build artefacts.
- Root configs (`package.json`, `tsconfig.json`) manage scripts, TypeScript options, and Plasmo manifest metadata.

## Build, Test, and Development Commands
- `pnpm install` (or `npm install`) — install dependencies; pnpm is faster and matches the lockfile.
- `pnpm dev` — run Plasmo in watch mode; refreshes `build/chrome-mv3-dev` for Chrome’s unpacked extension flow.
- `pnpm build` — produce an optimized production bundle in `build/chrome-mv3-prod`.
- `pnpm package` — zip the production bundle to `build/chrome-mv3-prod.zip` for Web Store uploads.

## Coding Style & Naming Conventions
- TypeScript + React with 2-space indentation; prefer arrow functions and hooks for stateful logic.
- Name components in `PascalCase`, hooks/utilities in `camelCase`, and CSS classes in `kebab-case`.
- Keep imports path-based (no absolute aliases) and group React imports before assets/styles.
- Run `pnpm build` before committing to catch TypeScript or Plasmo manifest issues.

## Testing Guidelines
- Automated tests are not yet configured; rely on manual verification.
- After `pnpm dev`, load `build/chrome-mv3-dev` via `chrome://extensions` → “Load unpacked” and confirm the play button opens `https://driftbossgame.org` in a new tab.
- For release checks, build with `pnpm build` and re-load `build/chrome-mv3-prod` to confirm no environment-specific regressions.

## Commit & Pull Request Guidelines
- Repository history is new; follow concise, imperative commit messages (e.g., `Add splash hover animation`).
- Include relevant context in PR descriptions: feature summary, manual test notes, and screenshots of the popup when UI changes.
- Link issues or tasks when available, request reviews early for UX changes, and ensure the packaged zip is regenerated when behaviour changes.

## Extension Packaging Tips
- Keep Chrome permissions minimal; update `package.json` manifest overrides if new APIs are required.
- Verify icon sizes remain under `assets/` so packaging scripts bundle them automatically.
- Before publishing, double-check the zipped build in `build/chrome-mv3-prod.zip` opens the correct Drift Boss URL and contains the latest assets.
